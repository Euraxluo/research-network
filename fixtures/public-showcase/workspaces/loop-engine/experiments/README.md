1. Convert a browser research run into a Research Asset package.
2. Replay the published loop from Walrus release files and Sui registry anchors.
3. Fork the asset into a second workspace while preserving graph edges and attribution.

Passing runs should produce a readable paper, installable skill, executable workflow, and live
registry links for object, transaction, blob, manifest, and commit.
