# Subagent 审查验收任务清单

维护日期：2026-06-18

本文件是 `06-acceptance-review-task-list.md` 的重跑版。用户已要求“每一个测试点都用 subagent 跑”，因此这里的规则是：**一行验收点绑定一个独立 subagent；没有对应 subagent 输出，不允许打勾**。

## 规则

- 状态只能由对应 subagent 或用户要求的主流程串行复验结果推进：`[ ] queued`、`[~] running`、`[x] pass`、`[!] fail`、`[blocked] blocked`。
- `pass` 只证明该行验收点的证据成立；本地测试通过不能替代真实 testnet、mainnet、Walrus、Seal、Indexer 或普通浏览器 receipt。
- 需要真实账号、资金、prover、salt、Walrus/Seal、GitHub OAuth 或 mainnet 配置的点，subagent 必须尝试审查可用前置，并在缺失时标为 `blocked`，不能用 mock/demo 顶替。
- subagent 输出不得包含 JWT、ephemeral secret、salt、GitHub token、Seal 明文、`.research-network/secrets/*` 原文。
- subagent 原始日志放在 `.research-network/acceptance/subagents/logs/`，不提交；本文只记录脱敏摘要和结果文件路径。
